# Google Photos Backup & Sync Bot

## Описание проекта
Локальный Python-скрипт для автоматизации резервного копирования фотографий и видео из Google Photos с последующей умной очисткой облака. 
Скрипт реализует стратегию "Горячего и холодного хранения": все новые и старые фото скачиваются на локальный диск, но из облака удаляются только файлы старше определенного порога (например, 180 дней), что позволяет сохранить их в галерее на телефоне для быстрого доступа.

## Технический стек
- **ОС:** Windows
- **Язык:** Python 3
- **Браузерная автоматизация:** `playwright` (Sync API)
- **Работа с медиа:** `Pillow` (чтение EXIF-тегов)
- **Стандартные библиотеки:** `zipfile`, `shutil`, `re`, `logging`, `datetime`, `os`, `json`, `hashlib`

## Структура путей
- **Рабочая директория:** Локальная папка проекта.
- **Профиль браузера (сессии, куки):** `PROFILE_DIR` (по умолчанию `bot_profile/`).
- **Временная папка (скачивание/распаковка):** `TEMP_DIR` (по умолчанию `temp_zip/`).
- **Целевая директория для бэкапа:** `DEST_DIR` (Задается в конфиге. Создает внутри папки вида `Фото Google [ГОД]`).
- **Файл состояния (манифест):** `STATE_FILE` (по умолчанию `backup_manifest.json`).
- **Логи:** `app.log`

## Настройки (config.json)
Скрипт автоматически создает и использует файл `config.json`:
- `DEST_DIR`: Целевая папка для сохранения архива.
- `PROFILE_DIR` / `TEMP_DIR` / `STATE_FILE`: Названия или пути к рабочим директориям/файлам.
- `DAYS_TO_KEEP_IN_CLOUD`: Возраст файлов в днях, после которого они разрешены к удалению из облака.
- `STOP_AT_DATE`: Дата в формате `YYYY-MM-DD`. Если скрипт доходит до этой даты, он завершает работу.
- `HEADLESS_MODE`: Булево значение (`true`/`false`). Запуск без визуального окна браузера.
- `ALLOW_CLOUD_DELETE`: Включает/выключает реальное удаление файлов из облака (нажатие кнопки корзины).
- `DRY_RUN_DELETE`: При `true` имитирует удаление в логах, но оставляет файлы в облаке.
- `MAX_DELETE_BLOCKS_PER_RUN`: Лимит удаления блоков за один запуск.
- `LOG_LEVEL`: Подробность логов: `WARNING` для старта/итога/проблем, `INFO` для скачиваний/удалений/dry-run, `DEBUG` для полной UI-механики и найденных блоков.
- `SCROLL_STEP_PIXELS`: Шаг прокрутки Google Photos. По умолчанию `500`, чтобы между экранами оставался безопасный нахлест.
- `SCROLL_WAIT_MS`: Пауза после прокрутки для подгрузки новых элементов.

## Основной алгоритм работы
1. **Инициализация браузера:** Playwright запускает Chromium с сохранением профиля, маскируясь под обычного пользователя (отключены флаги автоматизации `--enable-automation` и `AutomationControlled`).
2. **Авторизация:** Проверяется URL. Если требуется логин, скрипт дает пользователю 5 минут на ручной ввод.
3. **Поиск фото:** Скрипт ищет чекбоксы групп фото по дням (`aria-label="Выбрать ..."`).
4. **Оценка и идентификация:** Извлекается дата из `aria-label`. Рассчитывается возраст и генерируется SHA-256 `block_id`.
5. **Проверка по манифесту:**
   - Если блок подтвержден в `backup_manifest.json` и не требует удаления — он пропускается без взаимодействия с UI.
   - Если блок старше порога, но архив не подтвержден локально — скрипт попытается скачать его заново.
6. **Выделение (Умный клик):** Для обхода ограничений React Virtual DOM используется 3 попытки клика по статичному локатору (`force=True`), в крайнем случае применяется JS-клик.
7. **Скачивание:** Отправляется шорткат `Shift+D`. Файлы загружаются во временную папку.
8. **Локальная обработка и дедупликация:**
   - Извлечение из ZIP, игнорирование технических `.json`.
   - Сортировка по годам (`Фото Google [ГОД]`) на основе EXIF или имени файла.
   - Файлы верифицируются по SHA-256 хешу. Если файл с таким хешем уже существует, он дедуплицируется.
9. **Учет состояния (Идемпотентность):** 
   - Успешные скачивания фиксируются в `backup_manifest.json` (`files_verified=true`).
   - Загруженные групповые блоки записываются в `covered_dates`. При обнаружении одиночных фото этой даты, скрипт помечает их покрытыми группой и пропускает скачивание.
10. **Удаление или пропуск:**
   - Удаление из облака выполняется **только** если возраст превышает `DAYS_TO_KEEP_IN_CLOUD` и локальная копия надежно подтверждена манифестом.
   - Перед удалением проверяется фокус и наличие выделения на файлах.
   - Если файл "молодой" — нажимается `Escape` для снятия выделения, он остается в облаке.
11. **Пагинация (Умный скролл):** Скролл к последнему видимому элементу, наведение мыши в центр экрана и прокрутка колесиком с настраиваемым шагом `SCROLL_STEP_PIXELS` для плавной загрузки DOM без больших перепрыгиваний.
12. **Отчетность:** В конце выводится сводный лог сессии. Отдельно доступна утилита `manifest_report.py` для аналитики без запуска браузера.

## Ключевые решения и фиксы
- **Двойная проверка перед удалением:** Настройка `DAYS_TO_KEEP_IN_CLOUD` является лишь разрешением, но фактическое удаление блокируется, пока файлы не окажутся локально распакованными, проверенными по размеру/хешу и записанными в manifest.
- **Подтверждение удаления UI:** После клика подтверждения скрипт ждет закрытия модального окна и исчезновения исходного checkbox из DOM. Manifest помечается удаленным только после этой проверки.
- **Групповое покрытие (date coverage):** Решает проблему дублирования загрузки из-за особенностей виртуального DOM Google Photos, автоматически перекрывая дочерние одиночные элементы датами успешно скачанных групп.
- **Каскадное удаление в manifest:** При удалении группового блока дочерние item-записи, покрытые этой группой, тоже отмечаются удаленными через `deleted_with_block_id`.
- **Playwright Фокус:** Окно может терять фокус, поэтому перед хоткеями и кликами вызывается `page.bring_to_front()` и `page.locator('body').focus()`.
- **Восстановление битых загрузок:** Manifest является единственным доказательством сохранения. Если файла нет в манифесте, он будет скачан заново.
- **Защита от Virtual DOM:** Окно браузера жестко фиксируется в размере 1920x1080 для стабильной загрузки элементов в память. Клик производится не по плавающим элементам, а по статичным локаторам с ожиданием смены атрибута `aria-checked`.
