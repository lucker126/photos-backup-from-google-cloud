# 📸 Google Photos Backup & Sync Bot

## 📌 Описание проекта
Локальный Python-скрипт для автоматизации резервного копирования фотографий и видео из Google Photos с последующей умной очисткой облака. 
Скрипт реализует стратегию "Горячего и холодного хранения": все новые и старые фото скачиваются на жесткий диск, но из облака удаляются только файлы старше 180 дней (чтобы сохранить их в галерее на телефоне для быстрого доступа).

## 🛠 Технический стек
- **ОС:** Windows
- **Язык:** Python 3
- **Браузерная автоматизация:** `playwright` (Sync API)
- **Работа с медиа:** `Pillow` (чтение EXIF-тегов)
- **Стандартные библиотеки:** `zipfile`, `shutil`, `re`, `logging`, `datetime`, `os`

## 📂 Структура путей
- **Рабочая директория:** `E:\VSCode\photos-backup`
- **Профиль браузера (сессии, куки):** `E:\VSCode\photos-backup\bot_profile`
- **Временная папка (скачивание/распаковка):** `E:\VSCode\photos-backup\temp_zip`
- **Целевой диск для бэкапа:** `D:\` (По умолчанию. Создает папки вида `Фото Google [ГОД]`)
- **Логи:** `E:\VSCode\photos-backup\app.log`

## 🛠 Настройки (config.json)
Скрипт автоматически создает и использует файл `config.json` для масштабирования:
- `DEST_DIR`: Целевая папка для сохранения (по умолчанию `D:\`).
- `PROFILE_DIR` / `TEMP_DIR`: Пути к системным папкам (можно менять для запуска разных аккаунтов).
- `DAYS_TO_KEEP_IN_CLOUD`: Возраст файлов в днях, после которого они удаляются из облака (по умолчанию `180`).
- `SKIP_PERIOD_START` / `SKIP_PERIOD_END`: Даты в формате `YYYY-MM-DD`. Скрипт **автоматически обновляет** этот период после обработки каждого блока, чтобы при следующем запуске не скачивать файлы повторно.
- `STOP_AT_DATE`: Дата в формате `YYYY-MM-DD`. Если скрипт доходит до этой или более старой даты, он полностью завершает работу (идеально для еженедельного запуска).
- `HEADLESS_MODE`: Булево значение (`true`/`false`). Запуск без визуального окна браузера.

## ⚙️ Основной алгоритм работы
1. **Инициализация браузера:** Playwright запускает Chromium с сохранением профиля, маскируясь под обычного пользователя (отключены флаги `AutomationControlled` и `--enable-automation` для обхода защиты Google).
2. **Авторизация:** Проверяется URL. Если требуется логин (или 2FA), скрипт дает пользователю 5 минут на ручной ввод.
3. **Поиск фото:** Скрипт ищет чекбоксы групп фото по дням (`aria-label="Выбрать ..."`).
4. **Оценка возраста:** Извлекается дата из `aria-label` (например, "4 мая", "28 авг. 2023 г."). Возраст фото высчитывается в днях относительно текущей даты.
5. **Выделение (Умный клик):** Для обхода ограничений React Virtual DOM используется 3 попытки клика по статичному локатору (игнорируя состояние `aria-checked`). Если обычный `force=True` клик не срабатывает, применяется запасной JS-клик `evaluate("el => el.click()")`.
6. **Скачивание:** Отправляется шорткат `Shift+D`. Архив скачивается во временную папку.
7. **Дедупликация по датам:** Чтобы скрипт не скачивал одиночные фотографии из уже скачанной группы (из-за особенностей Virtual DOM), обработанные даты (YYYY-MM-DD) сохраняются в кэш и пропускаются при дальнейшей прокрутке.
8. **Обработка архива:**
   - Проверяется тип файла: скрипт умеет распаковывать ZIP-архивы и обрабатывать одиночные медиафайлы (если скачано всего 1 фото).
   - Игнорируются технические `.json` файлы.
   - Год создания фото/видео определяется в 3 этапа: 
     1) Из имени файла (например, `VID_2024...`); 
     2) Из EXIF (`DateTimeOriginal`); 
     3) Из текста `aria-label` (fallback).
   - Скрипт перемещает файлы в `D:\Фото Google [ГОД]`.
9. **Защита от дубликатов (Идемпотентность):** Если файл уже есть на диске `D:\` и его размер до байта совпадает со скачанным, скрипт пропускает его (защита от прерванных сессий). Если размер отличается — добавляет индекс `_1`, `_2` к имени.
10. **Удаление или пропуск:** 
   - **Восстановление выделения:** Если после скачивания (`Shift+D`) Google автоматически снимает выделение, скрипт повторно кликает на галочку перед удалением.
   - Если возраст группы **> 180 дней**: нажимается кнопка "В корзину", подтверждается в модальном окне.
   - Если возраст группы **<= 180 дней**: нажимается `Escape` (снятие выделения), фото остаются в облаке.
11. **Пагинация (Умный скролл):** Скрипт скроллит к последнему видимому элементу (чтобы заставить React подгрузить DOM), затем ставит курсор в центр экрана и крутит колесико мыши вниз.
12. **Сохранение состояния:** После каждой успешной операции (скачивание/удаление) скрипт обновляет `config.json`, записывая туда расширенный `SKIP_PERIOD`.
13. **Отчетность:** В конце работы выводится сводный отчет (размер обработанных данных, количество новых/удаленных файлов и их временные рамки).

## 🛡 Ключевые решения и фиксы (важно не сломать!)
- **Playwright Фокус:** Окно может терять фокус, поэтому перед хоткеями и кликами вызывается `page.bring_to_front()` и `page.locator('body').focus()`.
- **Проверка клика:** Скрипт проверяет успешность выделения ожиданием появления чекбокса с `aria-checked="true"`, так как панель корзины может перекрываться или рендериться с задержкой.
- **Невидимый режим:** Готов к переводу в `headless=True` (работает стабильно в фоне).
- **Обработка видео:** Pillow падает при попытке открыть `.mp4`, исключение перехватывается, и год парсится из названия или UI.
- **Защита от Virtual DOM:** Окно браузера жестко фиксируется в размере 1920x1080 для загрузки максимального количества элементов в память.

## Обновление: безопасное удаление и Git-подготовка
- `SKIP_PERIOD` больше не должен считаться доказательством того, что файлы реально сохранены локально. Он может оставаться только legacy-подсказкой.
- Новый источник истины для удаления из облака: `backup_manifest.json`.
- Блок можно удалить из Google Photos только если он старше `DAYS_TO_KEEP_IN_CLOUD` и имеет подтверждение `files_verified` в manifest.
- Если блок старше порога, но не подтвержден локально, скрипт сначала скачивает и проверяет его. При ошибке блок остается в облаке.
- После успешной групповой загрузки manifest записывает `covered_dates`: одиночные фото той же даты больше не скачиваются повторно, а считаются покрытыми проверенной группой.
- `config.json`, `backup_manifest.json`, `bot_profile`, `temp_zip`, `venv` и логи подготовлены к исключению из будущего Git-репозитория через `.gitignore`.
